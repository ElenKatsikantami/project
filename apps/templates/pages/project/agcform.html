Whats is AGS4 ?

AGS4 is a standard for transferring data between organizations in the site investigation and geotechnical/geoenvironmental industry, independent of software, hardware or operating system. Typical information that are being shared are ground investigation and laboratory test results and monitoring data as well as standard information on the project and machinery and standards used for the testing.

File Format

The AGS files are simple digital files (similar to .txt files) which store site investigation and geotechnical/environmental data. Data is structured in Groups. Each group has a unique name, headings, units and type to describe what information the data contain.
The main parts of an AGS file are:

  1. GROUP – indicates a collection of similar data items in the dictionary (spreadsheet tab)
  2. HEADING – indicates what information is contained at each attribute/column in a data group (column in a tab/group)
  3. UNIT – the units for each column/attribute
  4. TYPE – the attribute/column type for each data, ie. X for text or 2DP for two decimal places (the column formatting)
  5. DATA – one or more data rows conforming to the TYPE (the lines of data after fixed header). Each row of DATA can have multiple variables as per the HEADING.

The HEADING, UNIT and TYPE are collectively called the HEADER and define the main GROUP attributes. Each row in the data file must start with a DATA DESCRIPTOR that defines the contents of that row. Main Descriptors are GROUP, HEADING, UNIT, TYPE and DATA.
